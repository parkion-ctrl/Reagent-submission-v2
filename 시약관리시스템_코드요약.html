3. 데이터베이스 구조

3-1. 부서별 스키마 분리

부서스키마명포함 테이블
진단검사의학과dlabinventory, transaction_history
병리과pathinventory, transaction_history
핵의학과nminventory, transaction_history
유해물질hazinventory, transaction_history
로그인한 사용자의 부서가 미들웨어에서 자동으로 감지되어 해당 스키마의 search_path가 설정됩니다. 슈퍼유저는 상단 드롭다운으로 부서를 자유롭게 전환할 수 있습니다.

3-2. inventory 테이블 (시약 마스터 + 재고)

컬럼타입설명
idINTEGER IDENTITYPK
hazardousTEXT유해물질 여부 (Y/N)
partTEXT파트 코드 (HE/TA/BB/ML/IM/CO/PB/ZZ)
item_codeTEXT품목코드
item_nameTEXT품목명
lot_noTEXTLot 번호
expiry_dateTEXT유효기간 (YYYY-MM-DD)
spec / unitTEXT규격 / 단위
reagent_typeTEXT시약구분 (1~5)
equipment / vendorTEXT장비 / 업체
safety_stockINTEGER안전재고
current_stockINTEGER현재고
required_qtyINTEGER필요수량 (safety_stock - current_stock)
disposed_atTEXT폐기일 (NULL이면 현용)
disposal_reason / disposal_typeTEXT폐기 사유 / 유형
opened_at / parallel_atTEXT개봉일 / 병행검사일
base_item_nameTEXTOld/New lot 처리용 원본 품목명
lot_statusTEXTNEW / OLD / 빈값

6. 핵심 파일 설명

파일역할
app/core/db.pypsycopg 연결 래퍼. thread-local로 현재 스키마 관리. get_connection() 호출 시 자동으로 해당 부서 스키마의 search_path 적용. init_all_schemas()로 4개 스키마 테이블 자동 생성
lab/middleware.py모든 요청에서 로그인 사용자의 부서를 읽어 set_schema() 호출. 슈퍼유저는 세션의 superuser_active_dept 우선 적용, 기본값 진단검사의학과
lab/context_processors.py모든 템플릿에 user_department, is_dlab, is_superuser_session, superuser_dept_list 자동 주입
lab/views.py모든 뷰 함수. _get_part(request)로 진검 외 부서에서 파트를 ZZ로 강제. 관리자 전용 뷰는 can_access_admin_area()로 보호
app/services/reagent_history_service.pysync_expired_reagents(): 유효기간 만료 시약 자동 폐기 처리, Old/New lot 상태 동기화. 모든 조회 뷰에서 호출됨
app/services/history_service.pybackfill_remaining_stock(): 이력 페이지 로드 시 잔여 재고 재계산
app/utils/constants.pyPART_MAP (파트코드↔명칭), REAGENT_TYPE_MAP (1~5↔시약구분), DEPT_SCHEMA_MAP (부서명↔스키마명)
